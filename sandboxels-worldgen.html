Sandboxels worldgen

This page aims to describe how Sandboxels defines worldgen presets. This system is really complicated.

Terminology

Real number: A number that may have a decimal, such as 0.1 or 36.7.
Integer: A number without a decimal, such as 0 or 12.

baseHeight

baseHeight is a real number that nominally defines the height of the world's bottom relative to the height of the canvas. The default value is 0.5, which is assumed if it is not specified (due to how this is checked, a value of 0 is treated as an unspecified value and 0.5 is used.
Values above or below 0.5 squish the world upwards and stretch it downwards, respectively.


layers

layers is an array of arrays that describes the worldgen preset's layers and elements. Each array is of the format [real number, string] or [real number, string, real number].

First argument

The first item (hereafter bottom1) is a number, usually between 0 and 1.   bottom defines the bottom start of the layer, or the lowest position (relative to the vertical midpoint of the canvas) above which the layer's element will be placed.   If the bottom value is 0, it will be changed to a random negative number on world generation.
For instance, a bottom value of 0.5 will define a section extending, assuming a baseHeight of 0.5, from {1/4 above the bottom of the canvas} to {either the vertical midpoint or the bottom value of the next highest (i.e. the previous) layer}.
Layers are listed topmost first, with descending bottom values.   Any layers deviating from this order will override the layers after them.
A bottom value of 1 indicates the midpoint, and a value of 0 indicates the bottom of the canvas.   Values below 0 will try to generate pixels below the canvas.   Though these pixels do not generate because of the canvas edge, a negative value helps to fill in gaps at the bottom of the world caused by random offsets.   The extra material generated below the world will also generate (albeit compressed vertically) if baseHeight is greater than 0.5.
If two layers have the same decimal, the one that is defined first (higher) will apply; if the first layer with the decimal has a chance value less than 1, its pixels will have a corresponding random chance of being placed over the layer below.


The elements "red", "teal", "pink", "lime", and "orange" were modded in specifically for these images.

Second argument

The second item (hereafter element) is a string. This value defines the element (such as "dirt" or "iron") that comprises the layer.
element cannot be an array of element names. Elements that do not exist in the game will be ignored.


  1. The only layer value that is given any name in the code is the second value (element). The others are only referred to by their indices as layer[0] and layer[2].

This page previously stated that baseHeight values below 0.5 merely shift the world down. That is incorrect, and they actually stretch it downward.
